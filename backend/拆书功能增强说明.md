# 拆书功能增强说明

## 功能概述

本次更新为拆书功能添加了以下增强特性：

1. **Prompt模板数据库存储** - 将拆书功能的prompt存储到数据库中，支持多版本管理和自定义
2. **增强的AI分析** - 能够识别书中的角色、地图、每一章的大纲和细纲
3. **自动创建作品** - AI分析结果可以直接创建作品，包括角色、地点和章节

## 新增功能

### 1. Prompt模板管理

#### 数据库模型
- **PromptTemplate** 模型：存储prompt模板
  - `name`: 模板名称
  - `template_type`: 模板类型（如 `book_analysis`）
  - `prompt_content`: 提示词内容
  - `is_default`: 是否为默认模板
  - `variables`: 模板变量定义

#### API接口
- `POST /api/v1/prompt-templates/` - 创建prompt模板
- `GET /api/v1/prompt-templates/` - 获取模板列表
- `GET /api/v1/prompt-templates/{template_id}` - 获取模板详情
- `PUT /api/v1/prompt-templates/{template_id}` - 更新模板
- `DELETE /api/v1/prompt-templates/{template_id}` - 删除模板
- `GET /api/v1/prompt-templates/type/{template_type}/default` - 获取默认模板

### 2. 增强的拆书分析

#### 新增API接口
- `POST /ai/analyze-book?auto_create_work={true|false}` - 增强拆书分析接口

#### 功能特点
- 能够识别并提取：
  - **作品基本信息**：标题、描述、类型、标签等
  - **角色信息**：主要角色和次要角色，包括性格、外貌、背景等
  - **地点/地图信息**：故事中出现的所有地点及其描述
  - **章节信息**：每一章的大纲、细纲和完整内容

#### AI返回格式
AI会返回结构化的JSON数据，包含：
```json
{
  "work": {

  },
  "characters": [
    {
      "name": "角色名称",
      "description": "角色描述",
      "gender": "性别",
      "age": 年龄,
      "personality": {...},
      "appearance": {...},
      "background": {...},
      "is_main_character": true/false
    }
  ],
  "locations": [
    {
      "name": "地点名称",
      "description": "地点描述",
      "location_type": "city/forest/mountain等",
      "is_important": true/false
    }
  ],
  "chapters": [
    {
      "chapter_number": 章节号,
      "title": "章节标题",
      "summary": "章节概要",
      "outline": {
        "core_function": "核心功能",
        "key_points": ["关键点1", "关键点2"],
        "visual_scenes": ["画面1", "画面2"],

        "hook": "结尾钩子"
      },
      "detailed_outline": {
        "sections": [...]

      "content": "章节完整正文内容"
    }
  ]
}
```

### 3. 自动创建作品

当 `auto_create_work=true` 时，系统会：
1. 解析AI返回的JSON数据
2. 自动创建作品记录
3. 创建所有角色记录
4. 创建所有地点记录
5. 创建所有章节记录（包括内容）
6. 在ShareDB中创建章节文档

## 数据库变更

### 新增表

#### prompt_templates 表
```sql
CREATE TABLE prompt_templates (
    id SERIAL PRIMARY KEY,
    name VARCHAR(200) NOT NULL,
    description TEXT,
    template_type VARCHAR(50) NOT NULL,
    prompt_content TEXT NOT NULL,
    version VARCHAR(20) DEFAULT '1.0',
    is_default BOOLEAN DEFAULT FALSE,
    is_active BOOLEAN DEFAULT TRUE,
    variables JSON DEFAULT '{}',
    metadata JSON DEFAULT '{}',
    usage_count INTEGER DEFAULT 0,
    creator_id INTEGER REFERENCES users(id),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

#### locations 表
```sql
CREATE TABLE locations (
    id SERIAL PRIMARY KEY,
    work_id INTEGER REFERENCES works(id) ON DELETE CASCADE,
    name VARCHAR(100) NOT NULL,
    display_name VARCHAR(100),
    description TEXT,
    location_type VARCHAR(50),
    coordinates JSON DEFAULT '{}',
    image_url VARCHAR(255),
    parent_location_id INTEGER REFERENCES locations(id),
    tags JSON DEFAULT '[]',
    metadata JSON DEFAULT '{}',
    is_important BOOLEAN DEFAULT FALSE,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

### 模型更新

- `Work` 模型新增 `locations` 关系
- 新增 `PromptTemplate` 模型
- 新增 `Location` 模型

## 使用指南

### 1. 初始化默认Prompt模板

运行初始化脚本：
```bash
cd backend
python scripts/init_book_analysis_prompt.py
```

### 2. 前端调用增强分析

```typescript

  content,

  }
);

// 自动创建作品
const result = await analyzeBookEnhanced(
  content,
  true, // autoCreateWork
  (progress) => {
    
    if (progress.workResult) {
      
      // workResult包含: work_id, work_title, characters_count, locations_count, chapters_count
    }
  }
);
```

### 3. 管理Prompt模板

#### 创建模板
```bash
curl -X POST http://localhost:8001/api/v1/prompt-templates/ \
  -H "Content-Type: application/json" \
  -d '{
    "name": "自定义拆书模板",
    "template_type": "book_analysis",
    "prompt_content": "你的自定义prompt...",
    "is_default": false
  }'
```

#### 获取默认模板
```bash
curl http://localhost:8001/api/v1/prompt-templates/type/book_analysis/default
```

## 技术实现

### 后端服务

1. **BookAnalysisService** (`backend/src/memos/api/services/book_analysis_service.py`)
   - `get_default_prompt_template()`: 获取默认prompt模板
   - `get_enhanced_book_analysis_prompt()`: 获取增强的prompt内容
   - `parse_ai_response()`: 解析AI返回的JSON数据
   - `create_work_from_analysis()`: 从分析结果创建作品

2. **PromptTemplate模型** (`backend/src/memos/api/models/prompt_template.py`)
   - 存储和管理prompt模板

3. **Location模型** (`backend/src/memos/api/models/location.py`)
   - 存储地点/地图信息

### 前端API

1. **bookAnalysisApi.ts** 新增 `analyzeBookEnhanced()` 函数
   - 支持增强的拆书分析
   - 支持自动创建作品选项
   - 处理作品创建结果

## 注意事项

1. **AI模型要求**：增强分析需要较大的token限制，建议使用 `max_tokens` 至少 8000
2. **JSON格式**：AI必须返回有效的JSON格式，否则解析会失败
3. **章节内容**：每一章的 `content` 字段必须包含完整的章节正文
4. **权限**：自动创建作品需要用户认证，确保 `current_user_id` 可用
