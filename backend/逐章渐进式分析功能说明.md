# 逐章渐进式分析功能说明

## 功能概述

本次更新实现了逐章渐进式分析功能，解决了以下问题：

1. **Prompt模板管理** - 前端不再硬编码prompt，改为从后端数据库获取
2. **章节内容拼接** - 后端自动拼接每一章的内容，累积分析
3. **渐进式插入** - 每分析完一章就立即插入到目标作品中，包括角色、地点和章节

## 主要改进

### 1. Prompt模板从后端获取

#### 前端改进
- 移除了硬编码的 `getAnalysisPrompt()` 函数
- 新增 `getAnalysisPromptFromBackend()` 函数，从后端获取prompt模板
- 如果后端获取失败，使用默认prompt（向后兼容）

#### 后端改进
- `analyze_chapter` 接口自动从数据库获取默认prompt模板
- 如果没有提供prompt参数，会查询 `prompt_templates` 表获取 `chapter_analysis` 类型的默认模板

### 2. 逐章渐进式分析接口

#### 新增API接口
- `POST /ai/analyze-chapters-incremental?work_id={work_id}` - 逐章渐进式分析接口

#### 功能特点
- **自动分割章节**：根据章节标题模式（第X章、Chapter X等）自动分割内容
- **累积分析**：每分析一章时，会将之前所有章节的内容一起发送给AI，确保上下文连贯
- **即时插入**：每分析完一章，立即将结果插入到目标作品中

#### 工作流程
1. 接收包含多章内容的请求
2. 自动分割章节内容
3. 对每一章：
   - 累积当前章节和之前所有章节的内容
   - 调用AI分析（使用累积内容，确保上下文）
   - 解析AI返回的JSON数据
   - 立即插入到作品（角色、地点、章节）
4. 发送进度更新（章节开始、插入成功等）

### 3. 渐进式插入功能

#### 核心方法
`BookAnalysisService.incremental_insert_to_work()` - 渐进式插入分析结果

#### 插入逻辑

**角色处理（去重合并）**
- 检查角色是否已存在（按名称）
- 如果存在：更新现有角色信息（合并personality、appearance、background等）
- 如果不存在：创建新角色
- 自动合并tags，避免重复

**地点处理（去重合并）**
- 检查地点是否已存在（按名称）
- 如果存在：更新现有地点信息（合并metadata、tags等）
- 如果不存在：创建新地点
- 自动合并tags，避免重复

**章节处理（去重）**
- 检查章节是否已存在（按章节号）
- 如果存在：跳过（不重复创建）
- 如果不存在：创建新章节并保存内容到ShareDB

**作品信息更新**
- 仅在第一次分析时更新作品基本信息（description、genre、category、tags）
- 每次分析后更新章节数量统计

### 4. 章节内容分割

#### 分割方法
`BookAnalysisService.split_chapters_content()` - 自动分割章节内容

#### 支持的章节标题格式
- `第X章`、`第X回`（中文数字或阿拉伯数字）
- `Chapter X`（英文格式）
- `第零/一/二/三...章`（中文数字）

#### 分割逻辑
1. 使用正则表达式匹配章节标题
2. 根据匹配位置分割内容
3. 保留章节标题和内容
4. 如果没有找到章节标题，将整个内容作为一章

## API使用示例

### 前端调用示例

```typescript
import { analyzeChaptersIncremental } from '@/utils/bookAnalysisApi';

// 逐章渐进式分析
await analyzeChaptersIncremental(
  content, // 包含多章的内容
  workId,  // 目标作品ID
  (progress) => {
    if (progress.chapterIndex) {
      console.log(`分析第 ${progress.chapterIndex}/${progress.totalChapters} 章`);
    }
    if (progress.insertResult) {
      console.log('插入结果:', progress.insertResult);
      // {
      //   characters_processed: 5,
      //   locations_processed: 3,
      //   chapters_created: 1
      // }
    }
  }
);
```

### 后端API调用示例

```bash
curl -X POST "http://localhost:8001/ai/analyze-chapters-incremental?work_id=123" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer {token}" \
  -d '{
    "content": "第一章 开始\n\n这是第一章的内容...\n\n第二章 发展\n\n这是第二章的内容...",
    "settings": {
      "model": "gpt-4",
      "temperature": 0.7,
      "max_tokens": 8000
    }
  }'
```

## 流式响应消息类型

### 消息类型说明

1. **chapter_start** - 开始分析某一章
   ```json
   {
     "type": "chapter_start",
     "message": "开始分析第 1 章",
     "chapter_index": 1,
     "total_chapters": 10
   }
   ```

2. **chunk** - AI分析内容片段（标准SSE消息）

3. **chapter_inserted** - 章节插入成功
   ```json
   {
     "type": "chapter_inserted",
     "message": "第 1 章分析完成并已插入作品",
     "chapter_index": 1,
     "data": {
       "characters_processed": 5,
       "locations_processed": 3,
       "chapters_created": 1
     }
   }
   ```

4. **chapter_insert_error** - 章节插入失败
   ```json
   {
     "type": "chapter_insert_error",
     "message": "第 1 章插入失败: 错误信息",
     "chapter_index": 1
   }
   ```

5. **all_chapters_complete** - 所有章节分析完成
   ```json
   {
     "type": "all_chapters_complete",
     "message": "所有章节分析完成",
     "total_chapters": 10
   }
   ```

```python
chapter_pattern = r'(?:^|\n)(?:第[一二三四五六七八九十百千万\d]+[章节回]|Chapter\s+\d+|第\d+[章节回]|第[零一二三四五六七八九十]+[章节回])[^\n]*\n'
```

## 注意事项

1. **章节格式要求**：章节内容需要包含明确的章节标题，否则可能无法正确分割
2. **累积分析**：每章分析时会包含之前所有章节的内容，确保token限制足够
3. **去重策略**：角色和地点按名称去重，章节按章节号去重
4. **权限检查**：需要确保用户有编辑目标作品的权限
5. **错误处理**：单章插入失败不会影响其他章节的分析和插入
