# 前端同步逻辑问题分析

## 发现的问题

### 问题1：baseContent和content不匹配 ⚠️

**位置**：`syncDocumentState` 方法，第506-621行

**问题描述**：
1. 如果服务器版本更新（第506行），使用服务器内容作为baseContent（第516行）
2. 同时更新本地 `currentContent` 为服务器内容（第520行）
3. 但是，传入的 `content` 参数是用户当前编辑的内容，这个内容可能是基于旧版本的
4. 发送给服务器时：`base_content` 是新的服务器内容，但 `content` 是基于旧版本的，导致合并错误

**示例场景**：
1. 用户A编辑内容（版本1）→ 保存到本地
2. 用户B编辑内容（版本1→版本2）→ 保存到服务器
3. 用户A同步：
   - 获取服务器内容（版本2）→ 作为baseContent
   - 但用户A的content参数仍然是基于版本1的
   - 发送：`base_content=版本2内容, content=版本1内容` → 合并错误

**修复建议**：
```typescript
// 如果服务器版本更新，需要重新计算content
if (serverVersion > currentVersion) {
  baseContent = serverContent;
  this.currentVersion.set(documentId, serverVersion);
  this.currentContent.set(documentId, serverContent);
  
  // 关键修复：通知编辑器更新内容，等待编辑器更新后再同步
  // 或者：使用服务器内容作为baseContent，但content参数应该是基于服务器内容的
  // 或者：直接使用服务器内容，不发送用户的内容（因为用户的内容是基于旧版本的）
  
  // 方案1：通知编辑器更新，然后重新同步
  const callbacks = this.documentUpdateCallbacks.get(documentId);
  if (callbacks) {
    callbacks.forEach(callback => {
      callback(serverContent, serverVersion);
    });
  }
  
  // 等待编辑器更新后，使用更新后的内容重新同步
  // 或者：直接返回，让编辑器更新后再触发同步
  return {
    success: true,
    version: serverVersion,
    content: serverContent,
    operations: [],
    needsRefresh: true  // 标记需要刷新
  };
}
```

### 问题2：版本号使用不一致 ⚠️

**位置**：`syncDocumentState` 方法，第593-601行

**问题描述**：
1. 如果服务器版本更新，在第519行已经更新了 `currentVersion`
2. 但在第594行，仍然使用 `serverDoc.version || currentVersion`
3. 应该直接使用 `this.currentVersion.get(documentId)`，因为已经更新过了

**修复建议**：
```typescript
let syncVersion: number;
if (serverDoc) {
  // 使用已更新的版本号
  syncVersion = this.currentVersion.get(documentId) || serverDoc.version || 0;
} else if (fetchFailed) {
  syncVersion = Math.max(0, currentVersion - 1);
} else {
  syncVersion = currentVersion;
}
```

### 问题3：baseContent为空时的处理 ⚠️

**位置**：`syncDocumentState` 方法，第563-579行

**问题描述**：
1. 如果baseContent为空，从本地缓存获取
2. 但如果本地缓存也没有，baseContent仍然是空的
3. 发送给服务器时，`base_content` 为空，后端可能无法正确合并

**修复建议**：
```typescript
// 如果baseContent仍然为空，使用当前content作为baseContent
if (!baseContent) {
  baseContent = contentStr;  // 使用当前内容作为baseContent
  console.log('⚠️ [同步] baseContent为空，使用当前内容作为baseContent');
}
```

### 问题4：编辑器内容更新时机 ⚠️

**位置**：`syncDocumentState` 方法，第522-532行

**问题描述**：
1. 如果服务器版本更新，通知编辑器更新内容
2. 但是，同步操作会继续执行，使用旧的content参数
3. 应该等待编辑器更新后再同步，或者直接使用服务器内容

**修复建议**：
```typescript
if (serverVersion > currentVersion) {
  baseContent = serverContent;
  this.currentVersion.set(documentId, serverVersion);
  this.currentContent.set(documentId, serverContent);
  
  // 通知编辑器更新内容
  const callbacks = this.documentUpdateCallbacks.get(documentId);
  if (callbacks) {
    callbacks.forEach(callback => {
      callback(serverContent, serverVersion);
    });
  }
  
  // 关键修复：使用服务器内容作为content，因为用户的内容是基于旧版本的
  // 或者：等待编辑器更新后再同步
  // 方案：直接使用服务器内容，不发送用户的内容
  contentStr = serverContent;  // 使用服务器内容
}
```

## 正确的同步流程

### 理想流程

1. **获取服务器最新版本**
   - 如果服务器版本更新 → 使用服务器内容更新本地
   - 通知编辑器更新内容
   - 等待编辑器更新完成

2. **确定baseContent**
   - 如果服务器版本更新 → baseContent = 服务器内容
   - 否则 → baseContent = 上次同步的内容（currentContent）

3. **确定content**
   - 如果服务器版本更新 → content = 编辑器更新后的内容（需要等待）
   - 否则 → content = 用户当前编辑的内容

4. **发送同步请求**
   - base_content: 上次同步的内容
   - content: 当前编辑的内容
   - version: 当前版本号

## 修复方案

### 方案1：等待编辑器更新（推荐）

```typescript
if (serverVersion > currentVersion) {
  // 更新本地状态
  baseContent = serverContent;
  this.currentVersion.set(documentId, serverVersion);
  this.currentContent.set(documentId, serverContent);
  
  // 通知编辑器更新
  const callbacks = this.documentUpdateCallbacks.get(documentId);
  if (callbacks) {
    // 等待编辑器更新完成
    await Promise.all(callbacks.map(callback => 
      new Promise(resolve => {
        callback(serverContent, serverVersion);
        // 假设编辑器更新是同步的，或者需要等待
        setTimeout(resolve, 100);  // 给编辑器一些时间更新
      })
    ));
  }
  
  // 重新获取编辑器内容（更新后的）
  // 这里需要从编辑器获取最新内容，而不是使用传入的content参数
  // 或者：直接使用服务器内容，因为编辑器已经更新了
  contentStr = serverContent;  // 或者从编辑器获取
}
```

### 方案2：使用服务器内容（简单但可能丢失本地更改）

```typescript
if (serverVersion > currentVersion) {
  // 直接使用服务器内容，不发送用户的内容
  // 因为用户的内容是基于旧版本的，发送会导致合并错误
  return {
    success: true,
    version: serverVersion,
    content: serverContent,
    operations: [],
    needsRefresh: true
  };
}
```

### 方案3：重新计算差异（复杂但准确）

```typescript
if (serverVersion > currentVersion) {
  // 使用服务器内容作为baseContent
  baseContent = serverContent;
  
  // 用户的内容是基于旧版本的，需要重新计算差异
  // 或者：直接使用服务器内容，让用户重新编辑
  // 或者：尝试合并用户的内容到服务器内容
  
  // 这里需要知道用户编辑的原始内容（旧版本）
  // 然后计算从旧版本到新版本的差异，再应用到服务器内容
}
```

## 建议

**推荐使用方案1**，但需要：
1. 确保编辑器更新是同步的，或者可以等待
2. 或者：在编辑器更新后，重新触发同步
3. 或者：使用事件机制，等待编辑器更新完成后再继续
