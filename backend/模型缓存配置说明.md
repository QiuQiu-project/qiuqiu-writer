# 模型缓存配置说明

## 问题描述

每次启动服务时，系统都会尝试从 Hugging Face 重新下载模型，导致启动缓慢和网络超时。

## 终端常见警告说明

启动时可能看到以下日志，含义与处理方式如下：

| 日志 | 原因 | 处理 |
|------|------|------|
| `❌ missing NACOS_SERVER_ADDR / AK / SK / DATA_ID` | 未配置 Nacos 配置中心 | 不使用 Nacos 时可忽略 |
| `检测到 DeepSeek API，但 DeepSeek 不支持 embeddings API。自动回退到 sentence_transformer` | 当前 API 为 DeepSeek，DeepSeek 无 embedding 接口，已自动改用本地 sentence_transformer | 正常行为；若希望明确使用本地模型，可设置 `MOS_EMBEDDER_BACKEND=sentence_transformer` |
| `PydanticSerializationUnexpectedValue` | 部分配置在序列化时类型为 Pydantic 模型而非 dict | 不影响运行，可忽略 |
| `⚠️ Embedder backend mismatch! Config says 'sentence_transformer' but env expects 'universal_api'` | 环境变量默认是 `universal_api`，但实际因 DeepSeek 回退到了 `sentence_transformer` | 设置 `MOS_EMBEDDER_BACKEND=sentence_transformer` 即可消除（见下方） |

## 解决方案

### 1. 设置环境变量（推荐）

在启动服务前，设置以下环境变量：

```bash
# 设置 Hugging Face 缓存目录（可选，默认是 ~/.cache/huggingface）
export HF_HOME=~/.cache/huggingface

# 如果模型已下载，强制只使用本地缓存（避免网络检查）
export HF_LOCAL_FILES_ONLY=true
```

### 2. 在启动脚本中设置

修改启动脚本（如 `start.sh` 或 `start_ai_service.sh`），添加环境变量：

```bash
#!/bin/bash

# 设置模型缓存配置
export HF_HOME=~/.cache/huggingface
export HF_LOCAL_FILES_ONLY=true

# 启动服务
python -m uvicorn ...
```

### 3. 手动下载模型（首次使用）

如果网络不稳定，可以手动下载模型：

```bash
# 安装 huggingface_hub（huggingface-cli 来自该包，不是 huggingface-cli 包）
pip install huggingface_hub

# 必须用「当前项目 backend 的 venv」里的命令，否则会 command not found：
# 方式 A：先激活 backend 的 venv 再执行
cd backend
source .venv/bin/activate
huggingface-cli download nomic-ai/nomic-embed-text-v1.5

# 方式 B：直接用 backend/.venv 的完整路径
backend/.venv/bin/huggingface-cli download nomic-ai/nomic-embed-text-v1.5 --local-dir backend/models/nomic-embed-text-v1.5

# 或下载到默认缓存目录
backend/.venv/bin/huggingface-cli download nomic-ai/nomic-embed-text-v1.5 --cache-dir ~/.cache/huggingface
```

### 4. 检查模型缓存位置

模型缓存通常位于：
- Linux/Mac: `~/.cache/huggingface/hub/models--<model_name>`
- Windows: `C:\Users\<username>\.cache\huggingface\hub\models--<model_name>`

例如，`nomic-ai/nomic-embed-text-v1.5` 的缓存路径是：
```
~/.cache/huggingface/hub/models--nomic-ai--nomic-embed-text-v1.5
```

## 使用建议

### 首次启动（需要下载模型）
```bash
# 不设置 HF_LOCAL_FILES_ONLY，允许下载
python -m uvicorn ...
```

### 后续启动（使用缓存）
```bash
# 设置 HF_LOCAL_FILES_ONLY=true，跳过网络检查
export HF_LOCAL_FILES_ONLY=true
python -m uvicorn ...
```

## 验证缓存是否生效

启动服务时，查看日志：

```
✅ Model found in cache: ~/.cache/huggingface/hub/models--nomic-ai--nomic-embed-text-v1.5
✅ Successfully loaded SentenceTransformer model: nomic-ai/nomic-embed-text-v1.5
```

如果看到这些日志，说明缓存已生效，不会重新下载。

## 使用本地下载好的模型（推荐）

若希望**明确使用本地 embedder、避免 backend mismatch 警告**，可按以下方式配置。

### 方式一：使用 Hugging Face 缓存目录（推荐）

模型首次下载后会缓存在 `~/.cache/huggingface`（或 `HF_HOME` 指定目录），之后只需强制只用本地：

```bash
# 1. 明确指定使用本地 sentence_transformer，消除 "backend mismatch" 警告
export MOS_EMBEDDER_BACKEND=sentence_transformer

# 2. 可选：指定缓存目录（默认 ~/.cache/huggingface）
export HF_HOME=~/.cache/huggingface

# 3. 仅使用本地缓存，不访问网络
export HF_LOCAL_FILES_ONLY=true

# 4. 可选：指定模型（默认 nomic-ai/nomic-embed-text-v1.5）
export MOS_EMBEDDER_MODEL=nomic-ai/nomic-embed-text-v1.5

# 启动服务
./start_ai_api.sh
# 或
python -m uvicorn ...
```

### 方式二：使用本地目录路径

若模型已下载到**本地目录**（例如 `/data/models/nomic-embed-text-v1.5`），可直接把路径设为 `MOS_EMBEDDER_MODEL`：

```bash
export MOS_EMBEDDER_BACKEND=sentence_transformer
# 使用本地目录（绝对路径或相对路径均可）
export MOS_EMBEDDER_MODEL=/data/models/nomic-embed-text-v1.5
# 使用本地时建议关闭网络检查
export HF_LOCAL_FILES_ONLY=true

./start_ai_api.sh
```

本地目录需包含 SentenceTransformer 所需文件（如 `config.json`、`pytorch_model.bin` 等），可从 Hugging Face 下载后解压到该目录，或使用 `huggingface-cli download` 的 `--local-dir` 下载到指定目录：

```bash
# 下载到指定目录（可直接作为 MOS_EMBEDDER_MODEL）
huggingface-cli download nomic-ai/nomic-embed-text-v1.5 --local-dir /data/models/nomic-embed-text-v1.5
```

### 首次下载模型（仅需执行一次）

若尚未下载过模型，先允许网络访问完成下载：

```bash
# 允许下载（不要设 HF_LOCAL_FILES_ONLY=true）
export HF_LOCAL_FILES_ONLY=false
# 或直接不设置 HF_LOCAL_FILES_ONLY

# 下载到默认缓存
huggingface-cli download nomic-ai/nomic-embed-text-v1.5

# 或下载到指定目录
huggingface-cli download nomic-ai/nomic-embed-text-v1.5 --local-dir /data/models/nomic-embed-text-v1.5
```

下载完成后，再按「方式一」或「方式二」设置环境变量并启动服务即可。

## 流式输出用 Tokenizer（Qwen3-0.6B）

流式回复时用 tokenizer 做按 token 分块，默认从 Hugging Face 加载 `Qwen/Qwen3-0.6B`。和 nomic 一样可下载到 `backend/models` 后使用本地路径，避免联网。

### 下载到 backend/models（与 nomic 一致）

在**项目根目录**执行（使用 backend 的 venv）：

```bash
# 下载到 backend/models/Qwen3-0.6B
./backend/.venv/bin/huggingface-cli download Qwen/Qwen3-0.6B --local-dir ./backend/models/Qwen3-0.6B
```

或在 **backend 目录**下：

```bash
cd backend
.venv/bin/huggingface-cli download Qwen/Qwen3-0.6B --local-dir ./models/Qwen3-0.6B
```

### 使用本地路径

- **方式 A**：在 **backend/.env** 中设置（将路径改为你本机的 backend 绝对路径）：
  ```bash
  MOS_STREAMING_TOKENIZER_MODEL=/Users/pang/Documents/wawawriter/backend/models/Qwen3-0.6B
  ```
- **方式 B**：不设置该变量；若已存在 `backend/models/Qwen3-0.6B` 目录，启动脚本会优先使用该本地路径。

若无法访问 Hugging Face 且暂不下载，可禁用 tokenizer（退化为按字符分块）：
```bash
MOS_STREAMING_TOKENIZER_MODEL=
```
