# 文档同步功能数据库适配说明

## 概述

已为文档同步功能添加了数据库支持，创建了 `document_sync_history` 表用于记录文档同步历史和版本快照。

## 新增内容

### 1. 数据库模型

**文件**: `backend/src/memos/api/models/document.py`

新增 `DocumentSyncHistory` 模型，包含以下字段：
- `id`: 主键
- `document_id`: 文档ID（如 `work_4_chapter_6`）
- `version`: 同步后的版本号
- `content`: 同步后的内容（仅在 `create_version=True` 时保存）
- `content_hash`: 内容哈希值
- `user_id`: 同步用户ID
- `sync_type`: 同步类型（sync/version_snapshot/merge）
- `conflict_resolved`: 是否解决了冲突
- `merge_strategy`: 合并策略（smart_merge/diff_based/last_write_wins）
- `base_version`: 基础版本号
- `client_version`: 客户端版本号
- `server_version`: 服务器版本号
- `metadata`: 扩展元数据
- `created_at`: 创建时间

### 2. 模型关系更新

**文件**: `backend/src/memos/api/models/user.py`

在 `User` 模型中添加了与 `DocumentSyncHistory` 的关系：
```python
document_sync_history = relationship("DocumentSyncHistory", back_populates="user", cascade="all, delete-orphan")
```

### 3. ShareDB服务增强

**文件**: `backend/src/memos/api/services/sharedb_service.py`

在 `sync_document` 方法中添加了数据库历史记录保存功能：
- 自动计算内容哈希
- 记录合并策略和冲突解决信息
- 如果 `create_version=True`，保存内容快照
- 如果提供了 `db_session`，自动保存同步历史

### 4. 数据库初始化更新

**文件**: `backend/src/memos/api/core/database.py`

在 `init_db` 函数中添加了 `document` 模型的导入，确保新表会被创建。


### 如果表不存在

如果表不存在，运行初始化脚本会自动创建。如果初始化脚本失败，可以手动创建：

```sql
CREATE TABLE document_sync_history (
    id SERIAL PRIMARY KEY,
    document_id VARCHAR(255) NOT NULL,
    version INTEGER NOT NULL,
    content TEXT,
    content_hash VARCHAR(32),
    user_id INTEGER REFERENCES users(id),
    sync_type VARCHAR(20) DEFAULT 'sync',
    conflict_resolved BOOLEAN DEFAULT FALSE,
    merge_strategy VARCHAR(50),
    base_version INTEGER,
    client_version INTEGER,
    server_version INTEGER,
    metadata JSON DEFAULT '{}',
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 创建索引
CREATE INDEX idx_document_sync_history_document ON document_sync_history(document_id);
CREATE INDEX idx_document_sync_history_version ON document_sync_history(version);
CREATE INDEX idx_document_sync_history_user ON document_sync_history(user_id);
CREATE INDEX idx_document_sync_history_type ON document_sync_history(sync_type);
CREATE INDEX idx_document_sync_history_created ON document_sync_history(created_at);
CREATE INDEX idx_document_sync_history_doc_version ON document_sync_history(document_id, version);
CREATE INDEX idx_document_sync_history_doc_created ON document_sync_history(document_id, created_at);
```

## 使用说明

### 同步文档时保存历史

文档同步功能会自动保存同步历史（如果提供了数据库会话）。在 API 路由中，`db_session` 已经通过依赖注入提供：

```python
@router.post("/documents/sync", response_model=SyncResponse)
async def sync_document(
    request: DocumentSyncRequest,
    current_user_id: int = Depends(get_current_user_id),
    db: AsyncSession = Depends(get_async_db)  # 数据库会话已提供
):
    result = await sharedb_service.sync_document(
        document_id=request.doc_id,
        version=request.version,
        content=request.content,
        base_content=request.base_content,
        user_id=current_user_id,
        create_version=request.create_version,  # 如果为True，会保存内容快照
        db_session=db  # 传递数据库会话
    )
    return SyncResponse(**result)
```

### 查询同步历史

可以通过数据库查询同步历史：

```python
from memos.api.models.document import DocumentSyncHistory
from sqlalchemy import select

# 查询某个文档的同步历史
async with AsyncSessionLocal() as session:
    stmt = select(DocumentSyncHistory).where(
        DocumentSyncHistory.document_id == "work_4_chapter_6"
    ).order_by(DocumentSyncHistory.created_at.desc())
    
    result = await session.execute(stmt)
    history_records = result.scalars().all()
```

## 注意事项

1. **性能考虑**: 保存同步历史是异步的，如果保存失败不会影响同步功能本身
2. **内容存储**: 只有在 `create_version=True` 时才会保存完整内容，否则只保存元数据
3. **数据清理**: 建议定期清理旧的同步历史记录，避免数据库过大
4. **索引优化**: 已为常用查询字段创建了索引，确保查询性能
