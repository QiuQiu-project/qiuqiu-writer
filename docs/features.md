# 功能说明

## 用户系统

### 注册与邀请码

平台采用**邀请码制**注册，防止滥用。邀请码由管理员在后台生成，每个邀请码可设置使用次数限制。

### 个人资料

用户可编辑：昵称（display_name）、个人简介（bio）、头像（avatar_url）。

### 会话管理

- 登录颁发 Access Token（短期）+ Refresh Token（长期）
- Access Token 过期后，前端自动使用 Refresh Token 无感刷新
- 登出后 Token 加入黑名单（Redis），立即失效

## 作品管理

### 作品类型

| 类型 | 说明 | 专用编辑器 |
|------|------|-----------|
| `novel` | 小说 | 小说编辑器（NovelEditorPage） |
| `script` | 剧本 | 剧本编辑器（ScriptEditorPage） |

### 作品状态

| 状态 | 说明 |
|------|------|
| `active` | 正常状态，可编辑 |
| `published` | 已发布，对外可见 |
| `archived` | 已归档，只读 |

### 卷章结构

```
作品 (Work)
├── 卷 1 (Volume)
│   ├── 第 1 章 (Chapter)
│   ├── 第 2 章
│   └── 第 3 章
├── 卷 2
│   ├── 第 4 章
│   └── 第 5 章
└── 未分卷章节
    └── 番外
```

### 协作功能

- 作品所有者可邀请其他用户作为协作者
- 协作者权限可设置为：只读 / 可编辑
- 协作编辑通过 Yjs CRDT 实现无冲突同步

### 字数统计

章节保存时后端自动计算并更新字数，作品总字数为所有章节字数之和。

## 富文本编辑器

### 小说编辑器

基于 **TipTap** 构建，特性：
- 段落、标题、引用、分割线等基础格式
- @提及功能（@mention），可引用人物、场景等
- 字数实时统计
- 自动保存到本地（y-indexeddb）
- 支持 Markdown 快捷输入

### 剧本编辑器

专为剧本格式优化：
- 场景标题（Scene Heading）
- 动作描述（Action）
- 对白（Dialogue）
- 括号说明（Parenthetical）
- 转场（Transition）

---

## 实时协同编辑

### 工作原理

1. 每个章节对应一个 Yjs 文档（通过 `room_name` 唯一标识）
2. 浏览器通过 WebSocket 连接到后端 Yjs 服务
3. 所有编辑操作被编码为 Yjs CRDT 更新，广播给所有在线用户
4. 服务端将最新文档状态持久化到 PostgreSQL（Yjs 快照）
5. 文档内容同时通过 ShareDB 同步到 MongoDB

### 用户感知

- 可看到其他协作者的光标位置（Awareness）
- 离线后重连自动合并本地改动
- 无需手动解决冲突

### 版本历史

通过 **Yjs 快照** 实现版本管理：
- 手动或自动创建快照（可添加标签）
- 随时回溯到任意历史版本
- 支持两个版本之间对比（Diff）

## AI 辅助写作

### 章节续写

在编辑器中选中文本或将光标定位到段落末尾，调用 AI 续写功能。AI 会根据：
- 当前章节内容
- 作品背景设定
- 用户指令提示

生成符合风格的续写内容，以流式（SSE）方式逐字输出。

### 内容分析

对章节或整本书进行 AI 分析，分析维度包括：

| 分析类型 | 内容 |
|----------|------|
| 结构分析 | 起承转合、情节节奏 |
| 人物分析 | 人物性格一致性、弧光完整度 |
| 风格分析 | 文笔风格、语言特点 |
| 情节分析 | 伏笔、冲突、高潮点 |

### 提示词模板

管理员可在后台创建和管理**提示词模板**，支持：
- 模板变量（如 `{chapter_title}`、`{work_description}`）
- 组件化提示词（将多个子提示组合）
- 版本管理（修改历史可追溯）
- 按场景分类（章节分析、书籍分析、续写等）

### 记忆系统（可选）

启用 MemOS 后，AI 可以：
- 记住用户的写作偏好
- 在对话中引用之前讨论过的内容
- 基于作品设定提供个性化建议

配置方式见 [配置参考](./configuration.md#记忆系统)。

## 书籍分析

`book_analysis_service.py` 提供全面的书籍内容分析功能：

- **人物关系图谱** — 自动识别书中人物及其关系
- **世界观构建** — 提取地点、道具、规则体系
- **情节时间线** — 梳理故事发展脉络
- **写作建议** — 针对薄弱环节提供改进方向

## 内容广场

### 公开作品

作者可将作品设置为**公开**，显示在 `/novel` 或 `/script` 广场页供读者浏览。

### UGC 广场

`/ugc-plaza` 展示用户创作的多种形式内容，支持：
- 内容浏览与搜索
- 作者主页链接
- 分类筛选

---

## 主题切换

- 支持**深色**和**浅色**两套主题
- 用户偏好持久化存储（localStorage）
- 通过 `<html data-theme="dark|light">` 属性切换
- 页面加载时立即应用，防止闪烁（`initTheme()` 在 React 渲染前执行）

---

## 管理后台

### 系统监控

实时查看服务器资源使用情况：CPU、内存、磁盘占用。

### 用户管理

- 查看所有注册用户列表（含 ID、注册时间、状态）
- 启用/禁用用户账号
- 重置用户密码

### 邀请码管理

- 批量生成邀请码
- 设置邀请码有效期和使用次数
- 查看邀请码使用情况

### 提示词模板管理

- 在线编辑 AI 提示词模板，无需重启服务即时生效
- 模板版本控制，支持回滚

### 反馈管理

- 查看用户提交的反馈
- 更新反馈处理状态（待处理 / 处理中 / 已完成 / 已关闭）

### 审计日志

记录所有关键操作（用户注册、删除作品、权限变更等），保障安全可追溯。
